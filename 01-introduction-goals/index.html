    مقدمه و اهداف¶

    هدف سند¶

    این سند، به عنوان مرجع اصلی معماری سامانه فروشگاه اینترنتی تدوین شده است.

    هدف از نگارش این سند، ترسیم یک نقشه راه فنی جامع و قابل اتکا برای تمامی ذی‌نفعان پروژه است. این سند فراتر از یک توصیف صرف از ساختار سامانه، به تبیین منطق و دلایل پشت هر تصمیم معماری می‌پردازد و چارچوبی برای تصمیم‌گیری‌های آتی فراهم می‌آورد.

    محدوده و مرزهای سامانه¶

    این مستند، معماری یک پلتفرم تجارت الکترونیک را پوشش می‌دهد که به عنوان یک اکوسیستم دیجیتال، تعاملات شش گروه کاربری اصلی را مدیریت می‌کند:

    1. مشتریان که به جستجو، خرید، پرداخت و پیگیری سفارشات خود می‌پردازند.

    2. فروشندگان که مسئولیت ثبت، مدیریت موجودی و قیمت‌گذاری محصولات خود را بر عهده دارند.

    3. مدیران سامانه که بر عملکرد کل سامانه، مدیریت کاربران و تأیید نهایی فرآیندها نظارت دارند.

    4. تحلیل‌گران کسب‌وکار که به داده‌های حاصل از عملکرد سامانه برای استخراج بینش و گزارش‌های راهبردی دسترسی دارند.

    5. مهندسان دواپس که مسئولیت مدیریت استقرار، پایش و زیرساخت سامانه را بر عهده دارند.

    مرز این سامانه، تعاملات با سامانه‌های خارجی از جمله درگاه‌های پرداخت، سامانه‌های لجستیک، خدمات ارسال پیام، شبکه توزیع محتوا و سامانه‌های پایش خارجی را نیز در بر می‌گیرد.

    مخاطبان سند¶

    این سند برای طیف گسترده‌ای از مخاطبان با سطوح مختلف دانش فنی نگارش شده است:

    • توسعه‌دهندگان و معماران نرم‌افزار برای درک ساختار، تصمیمات فنی و چگونگی پیاده‌سازی

    • مدیران ارشد فنی برای ارزیابی و تأیید رویکردهای معماری و تخصیص منابع

    • مدیران پروژه برای برنامه‌ریزی، زمان‌بندی و مدیریت ریسک‌ها

    • کارشناسان حوزه کسب‌وکار برای درک کلان از قابلیت‌ها و محدودیت‌های سامانه

    ساختار معماری سامانه¶

    سامانه فروشگاه اینترنتی بر اساس معماری ریزخدمات¹ طراحی شده است. این معماری شامل اجزای زیر می‌باشد:

    برنامه‌های کاربردی:

    • برنامه وب با استفاده از Next.js که رابط کاربری برای مشتریان، مدیران و فروشندگان را فراهم می‌کند.

    • برنامه موبایل با استفاده از React Native که تجربه کاربری بومی را برای مشتریان ارائه می‌دهد.

    زیرساخت ارتباطی و امنیت:

    • شبکه توزیع محتوا با استفاده از Cloudflare برای توزیع محتوای ایستا و محافظت در برابر حملات انکار سرویس.

    • دروازه رابط برنامه‌نویسی با استفاده از Kong برای مسیریابی درخواست، احراز هویت و محدودیت نرخ.

    سرویس‌های حوزه کسب‌وکار:

    • سرویس احراز هویت با استفاده از Spring Boot و Keycloak برای مدیریت هویت، احراز هویت و کنترل دسترسی مبتنی بر نقش.

    • سرویس کاتالوگ با استفاده از Spring Boot و MongoDB برای مدیریت محصولات و دسته‌بندی‌ها.

    • سرویس موجودی با استفاده از Spring Boot، PostgreSQL و Redis برای مدیریت موجودی و رزرو.

    • سرویس سفارش با استفاده از Spring Boot و PostgreSQL برای مدیریت چرخه حیات سفارشات.

    • سرویس پرداخت با استفاده از Spring Boot برای پردازش پرداخت و یکپارچه‌سازی با درگاه‌های خارجی.

    • سرویس فروشنده با استفاده از Spring Boot و PostgreSQL برای مدیریت، تأیید و عملکرد فروشندگان.

    سرویس‌های پشتیبانی و تحلیل:

    • سرویس جستجو با استفاده از Elasticsearch برای جستجوی متن کامل و تکمیل خودکار.

    • سرویس توصیه‌گر با استفاده از Python و FastAPI برای ارائه توصیه‌های شخصی‌سازی‌شده.

    • سرویس اطلاع‌رسانی با استفاده از Spring Boot برای اطلاع‌رسانی چندکاناله.

    • سرویس تحلیل با استفاده از Spring Boot و ClickHouse برای گزارش‌دهی و تحلیل داده.

    زیرساخت داده و پایش:

    • کافکا برای پخش رویداد و ارتباطات ناهمگام.

    • حافظه نهان توزیع‌شده با استفاده از Redis.

    • پشته پایش با استفاده از Prometheus، Grafana، Jaeger و ELK برای مشاهده‌پذیری و پایش.

    • لایه مقاوم‌سازی با استفاده از Resilience4J و Istio برای افزایش تاب‌آوری سامانه.

    • سرویس پشتیبان‌گیری با استفاده از Spring Boot و AWS S3 برای پشتیبان‌گیری خودکار و بازیابی در شرایط فاجعه.

    پانویس‌ها¶

    ¹ ریزخدمات رویکردی در معماری نرم‌افزار که سیستم را به مجموعه‌ای از سرویس‌های کوچک، مستقل و قابل استقرار جداگانه تقسیم می‌کند. هر سرویس حول یک قابلیت کسب‌وکار خاص سازماندهی می‌شود و از طریق رابط‌های برنامه‌نویسی با سایر سرویس‌ها ارتباط برقرار می‌کند.
